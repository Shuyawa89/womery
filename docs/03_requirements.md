# 要件定義書

## 概要

本ドキュメントは、**Womery（ウーメリー）** の要件を定義する。

## プロダクト名

**Womery** - Working Memory（ワーキングメモリ）から派生。ADHDの人が苦手とするワーキングメモリを外部化し、補完するツール。

## 最終更新日

2024年12月18日

---

## プロダクトビジョン

### 解決したい課題

ADHDを持つ人は、日常の些細な選択（服選び、持ち物確認、次に何をするか）に無意識にリソースを消費し、本来集中すべき仕事や活動に使えるエネルギーが減少してしまう。

### ソリューション

日常の選択を極限まで減らす「外付け脳」を提供する。具体的には、時間ベースのルーティン管理、持ち物管理、即時メモ機能を通じて、「次に何をすべきか」を考える負担を軽減する。

### コンセプト：「プロADHD」を支援する

> 「未来の自分を一切信用しない」
> 「脳のメモリを記憶に使わない、全て外部化する」

このプロダクトは、ADHDの弱点を「意志力」ではなく「仕組み」でカバーすることを目指す。

### ターゲットユーザー

- ADHDを持つ社会人
- 朝の準備に時間がかかる人
- 忘れ物が多い人
- 日常のルーティン化で生産性を上げたい人

---

## 設計原則

本プロダクトの設計において、以下の原則を最優先とする。

### 1. 即時性（Speed First）

```
「忘れる前に記録する」がモットー。
入力の速さ > 記録の完全さ
```

- アプリを開いて **3タップ以内** でメモ/タスク登録が完了すること
- 「後で詳細を入力」ではなく「今この瞬間に記録完了」が原則
- 分類や整理は後から行う（または行わなくてもよい）

### 2. 認知負荷の最小化

```
「選択させない、考えさせない」
```

- ユーザーに選択肢を提示しない、または1つに絞る
- デフォルト値を賢く設定し、変更は任意とする
- 「何を入力すべきか」を迷わせるUIを避ける

### 3. 外部化の徹底

```
「脳に記憶させない、全てをシステムに預ける」
```

- 思いついたことは全て記録できる受け皿を用意
- 記録したものは適切なタイミングで提示される
- ユーザーが「思い出す」必要がない設計

## ユーザーストーリー

### MVP（最小限の実用可能なプロダクト）

#### US-001: 朝のルーティン自動提示

```
ADHDを持つユーザーとして、
朝起きたら「今日やるべきこと」が時間割形式で自動表示されてほしい。
なぜなら、朝に「何をすべきか」を考えること自体が負担だから。
```

**受け入れ条件:**

- 当日のルーティンが時間順に表示される
- 各ルーティン項目に開始時刻と終了時刻が表示される
- 現在時刻に該当するルーティンがハイライトされる

#### US-002: 出社/在宅によるルーティン切り替え

```
ADHDを持つユーザーとして、
前日に出社か在宅かを選択したら、翌日はそれに応じたルーティンが表示されてほしい。
なぜなら、出社日と在宅日で準備内容や時間が異なるから。
```

**受け入れ条件:**

- 「出社」「在宅」を選択できる
- 選択は前日に行う（翌日分の設定）
- 出社用ルーティンと在宅用ルーティンを別々に定義できる
- 当日は選択済みのルーティンが自動で表示される

#### US-003: 持ち物チェックリスト

```
ADHDを持つユーザーとして、
出社時に必要な持ち物のチェックリストが表示されてほしい。
なぜなら、忘れ物をしやすく、家を出る前に確認したいから。
```

**受け入れ条件:**

- 出社日のみ持ち物チェックリストが表示される
- チェックリストの項目を事前に登録できる
- チェック済み/未チェックを切り替えられる
- 全項目チェック完了時に視覚的フィードバックがある

#### US-004: タスクの即時入力

```
ADHDを持つユーザーとして、
思いついたタスクをすぐに入力できるようにしたい。
なぜなら、後で思い出そうとしても忘れてしまうから。
```

**受け入れ条件:**

- アプリを開いて3タップ以内でタスク登録が完了する
- タスク名のみで登録可能（詳細は任意）
- 登録後、元の画面に戻る
- 登録したタスクは「タスク整理」時間帯に確認できる

#### US-008: 即時メモ（分類不要の記録）

```
ADHDを持つユーザーとして、
思いついたことを「分類を考えずに」即座に記録したい。
なぜなら、「これはタスクか？メモか？」と考えている間に忘れてしまうから。
```

**受け入れ条件:**

- タスクかメモかを選択せずに記録できる（統一された入力口）
- 最低限「テキスト」だけで記録完了
- 分類は後から行う（または行わなくてもよい）
- Inbox的な場所に全て蓄積される

**設計メモ:**
US-004（タスク入力）とUS-008（即時メモ）は、UIとしては統合して「クイック入力」として実装する可能性あり。ドメインモデル設計時に検討する。

#### US-009: 翌日の準備完了確認（条件付きMVP）

```
ADHDを持つユーザーとして、
寝る前に「明日の準備が完了したか」をチェックリストで確認したい。
なぜなら、朝になってから慌てて準備すると忘れ物が増えるから。
```

**受け入れ条件:**

- 翌日の勤務形態（出社/在宅）が選択されていること
- 翌日の勤務形態に応じた準備項目が表示される
- チェック完了で「準備OK」の状態になる

**実装判断:**
シンプルに実装できる場合はMVPに含める。複雑になる場合はPhase 2に延期。

### Phase 2（MVP後）

#### US-005: Push通知によるリマインド

```
ADHDを持つユーザーとして、
ルーティンの開始時刻や持ち物確認のタイミングで通知が来てほしい。
なぜなら、時間の感覚が曖昧で、気づいたら遅れていることが多いから。
```

#### US-007: 物の定位置管理

```
ADHDを持つユーザーとして、
持ち物ごとに「定位置（置き場所）」を登録し、出発前に確認したい。
なぜなら、物を探す時間と「どこに置いたっけ」という認知負荷を減らしたいから。
```

### 将来機能

#### US-006: スマートホーム連携

```
ADHDを持つユーザーとして、
ルーティン開始時に照明やエアコンが自動で動いてほしい。
なぜなら、「起きる」「電気をつける」という選択すら減らしたいから。
```

**想定連携先:** Home Assistant

### 機能詳細

#### F-001: ルーティン表示

**概要:** 当日のルーティンを時間割形式で表示する。

**詳細仕様:**

- 時間軸は縦方向（上から下へ時間が進む）
- 各ルーティン項目は以下の情報を表示:
  - 開始時刻
  - 終了時刻
  - ルーティン名
  - （任意）アイコン
- 現在時刻に該当するルーティンをハイライト表示
- 過去のルーティンはグレーアウト

#### F-002: ルーティン登録

**概要:** ルーティンのテンプレート（出社用・在宅用）を作成・編集する。

**詳細仕様:**

- ルーティンテンプレートは「出社用」「在宅用」の2種類
- 各テンプレートに複数のルーティン項目を登録可能
- ルーティン項目の属性:
  - ルーティン名（必須）
  - 開始時刻（必須）
  - 終了時刻（必須）
  - 順序（ドラッグ&ドロップで並び替え）

#### F-003: 勤務場所選択

**概要:** 翌日の出社/在宅を選択する。

**詳細仕様:**

- 選択肢: 「出社」「在宅」
- 選択タイミング: 前日（任意のタイミングで変更可能）
- 未選択の場合のデフォルト: 前回と同じ、または在宅

#### F-007: クイック入力

**概要:** 思いついたタスク/メモを即座に登録する。

**詳細仕様:**

- 入力項目:
  - テキスト（必須） - タスク名またはメモ内容
- 最小限の入力でも登録可能（テキストのみ）
- フローティングボタン等で常にアクセス可能
- **3タップ以内で登録完了**を必須要件とする
- 登録されたものは全てInboxに入る

#### F-008: Inbox表示

**概要:** 未整理のタスク/メモを一覧表示する。

**詳細仕様:**

- 登録日時の降順で表示（新しいものが上）
- 各項目に対して以下の操作が可能:
  - 削除
  - タスクとしてマーク（将来的に期限設定など）
  - 完了としてマーク

## 非機能要件

### パフォーマンス

| 項目 | 要件 |
|------|------|
| 画面表示速度 | 初期表示 3秒以内 |
| API応答時間 | 95パーセンタイルで500ms以内 |
| クイック入力 | 登録完了まで3タップ以内 |

### 可用性

| 項目 | 要件 |
|------|------|
| 稼働率 | 99%以上（個人利用のため厳密でなくてよい） |
| 計画停止 | 事前通知なしで可（個人利用のため） |

### セキュリティ

| 項目 | 要件 |
|------|------|
| 認証 | 必須（個人データを扱うため） |
| 通信 | HTTPS必須 |
| データ保護 | 個人のルーティン・タスク情報を保護 |

### ユーザビリティ

| 項目 | 要件 |
|------|------|
| 対応デバイス | iPad、スマートフォン（iOS/Android） |
| レスポンシブ対応 | 必須 |
| オフライン対応 | Phase 2以降で検討 |

## 用語集（ユビキタス言語）

| 用語 | 英語 | 定義 |
|------|------|------|
| ルーティン | Routine | 毎日繰り返す一連の行動のテンプレート |
| ルーティン項目 | RoutineItem | ルーティンを構成する個々の行動（例：洗顔、朝食） |
| 勤務場所 | WorkLocation | 出社（Office）、在宅（Remote）、休日（DayOff）のいずれか |
| 日次スケジュール | DailySchedule | 特定の日付に対して適用されるルーティンと勤務場所の組み合わせ |
| 持ち物 | Belonging | 外出時に持っていくべき物品 |
| 持ち物チェック | BelongingCheck | 特定の日の持ち物確認状態 |
| クイックメモ | QuickMemo | 最小限の入力で即座に記録するメモ |
| Inbox | Inbox | 未整理のクイックメモが蓄積される場所 |
| タスク | Task | 具体的にやることが決定済みで、期日があり、実行する前提のもの |
| メモ | Memo | 将来やるかもしれないこと、アイデア、気になったことの記録 |
| タイムスロット | TimeSlot | 開始時刻と終了時刻で定義される時間枠 |
